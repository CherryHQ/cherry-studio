# 产品路线图 / 需求清单

更新时间：2026-06-07

## 北极星：这个产品最终想成为什么

一句话愿景：

> 让用户能够在 AI 回复的任意一点继续深挖，而不打断主线、不丢失上下文、不忘记自己最初要完成的目标。

更完整一点：

这个产品希望把传统线性 AI 聊天升级成一种 **可展开、可回溯、可组织、可总结的对话工作台**。  
用户不再只能一条一条往下聊，而是可以围绕 AI 回复中的某个词、某句话、某个知识点、某个步骤开启局部分支；主线继续保留，分支并列展开，最终形成一个类似 workflow / timeline / knowledge map 的结构化对话空间。

---

## 产品核心原则

1. **主线不能丢**
   - 用户始终知道自己最初在做什么；
   - 分支是对主线的局部展开，不应该把用户带离主线。

2. **局部可以深挖**
   - AI 回复中的任意一句话、一个术语、一个步骤，都可以继续追问；
   - 追问应该围绕选中内容回答，而不是重新泛泛回答。

3. **视觉上要可定位**
   - 源文本必须有高亮；
   - 分支和源文本之间要有明确视觉关系；
   - 用户能看出“这个 branch 是从哪里来的”。

4. **对话要能被整理**
   - 每个 branch 应该可以折叠、总结、归档；
   - 整轮对话应该有总结；
   - 长期要支持标签、Workspace、时间线、搜索。

5. **工程上小步推进**
   - 先做可验证 MVP；
   - 不一次性做大而全；
   - 每次只解决一个明确问题；
   - 重要工程过程必须沉淀到 `Chat APP-Docs`。

## 已完成

### 1. Text Anchor Branch：文本锚点分支 MVP

- 用户可以选中 assistant 回复中的一段文字；
- 右键菜单可以触发 branch 操作；
- branch 在当前界面右侧 side-by-side 展开；
- 主对话不跳走；
- branch AI 回复会围绕 selectedText 和 follow-up 聚焦回答。

### 2. Side-by-side Branch Panel

- 主聊天保留在左侧；
- 右侧出现 Branch Panel；
- Branch Panel 可以显示 selected passage；
- Branch Panel 可以显示 branch 对话流；
- branch 不再以普通 topic 页面跳转方式打开。

### 3. Hidden Prompt：隐藏上下文模板

- 系统会把 selectedText 和用户 follow-up 拼成隐藏上下文；
- 用户界面不直接暴露冗长系统模板；
- AI 回复能更聚焦于选中内容，而不是像重新开聊。

### 4. Branch Regenerate / Edit / Delete 保持焦点

- branch 内 regenerate 不会丢掉 selected passage 的上下文；
- 已通过 BranchAssistantContext 等方式保持 branch 局部语义；
- branch 内操作和主线操作初步解耦。

### 5. 可拖拽 Branch Panel

- 右侧 Branch Panel 可以拖拽调整宽度；
- 主聊天区域会自动 reflow；
- session 内保留上次拖动宽度；
- 刷新后恢复默认宽度。

### 6. 源选区高亮

- CSS Custom Highlight API 在当前环境不可见，已弃用；
- 当前采用 `span.branch-anchor-highlight` wrap 方案；
- 源文本高亮可见；
- A → B 切换不会累积高亮；
- 关闭 branch 后真实 span 可以清理到 0；
- 空 span 残留问题已修复。

### 7. 基础 Debug / Baseline 修复

- 修复 default assistant sentinel 导致 topic 创建失败的问题；
- 修复 assistant message modelId FK 问题；
- 修复 Ollama provider 识别 / Chat picker v2 数据源问题；
- 修复 assistant 回复完成后 UI 一直 loading 的问题；
- 修复 branch 相关上下文和高亮链路中的多个实际问题。

### 8. 多分支共存（P1，已完成并入库 2026-06-01~03）

- 同一主对话可同时打开多个 branch，每个独立高亮配色 + 独立折叠 / 关闭；
- 右侧面板 accordion 手风琴布局（单滚动区、标题贴正文，修 B1 重叠）；
- conversation 态可继续追问（follow-up），路由到正确 branch topic；
- 统一 composer 键盘（Enter 发送 / Shift+Enter 换行 / IME 守卫）。

### 9. 关闭去向 disposition + 真停后端（P1-S3 + B6，已完成并入库 2026-06-04~07）

- 关闭按 disposition 路由：默认 **pending → 静默删 fork topic** / 「保留」**kept → 留库**；
- delete-after-settle：删除推迟到流式 finalize 落地后，避免 404；
- 关闭分支会**中止其流式回复**（B5）；
- **B6-A**：abort 取靶改按存活 abortController（不依赖会被过早标 success 的 `message.status`），流式中关分支现已**真停后端**（Ollama 请求 canceled）。
