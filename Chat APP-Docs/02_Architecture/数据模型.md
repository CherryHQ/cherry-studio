# Cherry Studio 聊天模块 — 数据模型

> v1 / v2 并存期。**v1 = Dexie + Redux**（即将删），**v2 = SQLite + Drizzle**（目标）。本文档把两边对照列出。

## 1. 概念关系

```
Assistant (1) ──┐
                ├─< (n) Topic ──< (n) Message ──< (n) Block
                │
                └─ settings, prompt, modelId, mcpServerIds, ...
```

- **会话载体 = Topic**（v1、v2 一致）。代码里不存在独立的 `Conversation` 类型。
- **Message** 通过 `topicId` 挂到 Topic；通过 `parentId`（v2）/ `askId`（v1）形成「同一问题多个回复」的分组。
- **Block** 是消息的内容单元（文本、代码、图片、工具调用…）；v1 独立表，v2 内联 JSON。

## 2. 类型对照

### 2.1 Message

**v1**（`src/renderer/src/types/newMessage.ts`，约 L186）：
```ts
interface Message {
  id: string
  role: 'user' | 'assistant' | 'system'
  assistantId: string
  topicId: string
  createdAt: string
  status: 'sending' | 'success' | 'error' | 'paused' | ...
  modelId?: string
  model?: Model
  blocks: MessageBlock['id'][]       // ← 关键：仅存块 ID 数组
  usage?: TokenUsage
  metrics?: MessageMetrics
  askId?: string                     // ← v1 的"同问题多回复"分组键
}
```

**v2**（`packages/shared/data/types/message.ts`，约 L404 `MessageSchema`）：
```ts
interface Message {
  id: string                          // UUID
  topicId: string                     // FK
  parentId: string | null             // ← v2 改为树结构
  role: 'user' | 'assistant' | 'system'
  data: MessageData                   // ← 关键：块内联 JSON
  searchableText: string              // ← FTS5 索引字段
  status: 'SUCCESS' | 'ERROR' | 'PAUSED'
  siblingsGroupId: number             // 0=正常，>0=多回复组
  modelId: string | null              // FK → userModel
  modelSnapshot: ModelSnapshot | null // 冻结当时的模型快照
  traceId: string | null
  stats: MessageStats | null
  createdAt: string
  updatedAt: string
}

interface MessageData {
  blocks: MessageDataBlock[]          // 块直接嵌入
}
```

### 2.2 Topic

**v1**（约 L271）：
```ts
interface Topic {
  id: string
  type: 'chat' | 'agent' | ...
  assistantId: string
  name: string
  createdAt: string
  updatedAt: string
  messages: Message[]                  // 内联消息数组
  pinned?: boolean
  prompt?: string
}
```

**v2**（`packages/shared/data/types/topic.ts`，约 L18 `TopicSchema`）：
```ts
interface Topic {
  id: string                           // UUID
  name: string
  isNameManuallyEdited: boolean
  assistantId: string | null           // FK，可空（孤立 topic）
  activeNodeId: string | null          // 当前选中消息（树形导航）
  groupId: string | null               // FK → group
  orderKey: string                     // fractional index
  createdAt: string
  updatedAt: string
}
```

### 2.3 Assistant

**v2**（`packages/shared/data/types/assistant.ts`，约 L113 `AssistantSchema`）：
```ts
interface Assistant {
  id: string                           // UUID
  name: string
  prompt: string                       // 系统提示
  emoji: string | null
  description: string | null
  settings: AssistantSettings          // 温度、context、工具开关 …
  modelId: string | null
  mcpServerIds: string[]
  knowledgeBaseIds: string[]
  tags: string[]
  modelName: string | null
  createdAt: string
  updatedAt: string
}
```

**v1 → v2 关键变化：**
- v1 有保留字符串 `'default'` 作为系统 sentinel；**v2 取消**，UI 层运行时合成默认助手。
- v1 `assistant.topics: Topic[]` 内联；v2 完全分表，靠 FK 关联。

### 2.4 MessageBlock（块）

12 种 BlockType（v1、v2 一致）：

| BlockType | 用途 |
|---|---|
| MAIN_TEXT | 主文本（含 Markdown） |
| THINKING | 推理过程（Claude/o-series 思考链） |
| CODE | 独立代码块（一般已嵌入 MainText） |
| IMAGE | 单张图片 |
| FILE | 文件元数据 |
| TOOL | MCP 工具调用 + 结果 |
| CITATION | 引用列表（**v2 @deprecated**，已并入 `MainText.references`） |
| TRANSLATION | 翻译块 |
| VIDEO | 视频 |
| ERROR | 错误信息 |
| COMPACT | `/compact` 命令产物 |
| UNKNOWN | 占位（流式加载中） |

**v1 vs v2 存储差异：**
| 维度 | v1 | v2 |
|---|---|---|
| 表 | `db.message_blocks`（独立行） | 无独立表，嵌入 `message.data.blocks[]` |
| 挂载字段 | `Message.blocks: string[]` (ID 数组) | `Message.data.blocks: MessageDataBlock[]` |
| 引用 | 独立 `CitationMessageBlock` | 内联 `MainTextBlock.references: ContentReference[]` |
| 类型枚举 | `MessageBlockType`（v1） | `BlockType`（v2） |

## 3. SQLite Schema 摘要（v2）

### `message`（`src/main/data/db/schemas/message.ts`）
```
id              UUID PK
parentId        text (self FK, 树结构)
topicId         UUID FK → topic (CASCADE)
role            'user'|'assistant'|'system'
data            JSON<MessageData>
searchableText  text (FTS5 源)
status          'SUCCESS'|'ERROR'|'PAUSED'
siblingsGroupId int
modelId         text FK → userModel
modelSnapshot   JSON
traceId         text
stats           JSON
createdAt       timestamp
updatedAt       timestamp
```

附属虚拟表：
- `message_fts`（contentless FTS5 over `searchableText`，tokenize=trigram）
- INSERT / UPDATE / DELETE 触发器自动同步

### `topic`（`src/main/data/db/schemas/topic.ts`）
```
id                   UUID PK
name                 text
isNameManuallyEdited boolean
assistantId          text FK → assistant (SET NULL)
activeNodeId         text
groupId              text FK → group (SET NULL)
orderKey             fractional-index
createdAt            timestamp
updatedAt            timestamp
```

### `assistant`、`userModel`、`file`
（详见 `src/main/data/db/schemas/`，命名直观，本文档不再展开。）

## 5. v1 → v2 关键映射（迁移器视角）

来自 `src/main/data/migration/v2/migrators/ChatMigrator.ts`：

| v1 状态 | v2 目标 |
|---|---|
| `Topic.messages[]` 内联 | 拆为 `message` 表行（按 `topicId`） |
| `Message.blocks: string[]`（块 ID 数组） | 内联到 `message.data.blocks[]` |
| `Message.askId` + `foldSelected` | `parentId` + `siblingsGroupId` |
| 独立 `CitationMessageBlock` | 合并入 `MainTextBlock.references` |
| `Topic.assistantId === 'default'` | UUID（由 `AssistantMigrator` 解析） |
| 孤立 Topic（找不到助手） | `assistantId = NULL` |
| `MessageBlock.file: FileMetadata` | `file_ref` 表行（`sourceType='chat_message', sourceId=messageId`） |

迁移器同时处理：循环引用检测、孤立节点回收、时间戳一致性、askId → siblingsGroup 重新编号。

## 7. 一句话总结

**v2 数据模型已就位**（schema + 类型 + 迁移器都齐了），但 **UI 仍读 Redux + Dexie**，所以「类型对照表」中的两侧目前**都在线上活跃使用**。新开发若涉及聊天数据落地，**必须同时考虑 v1 路径**（不要绕过 messageThunk 直接写 DataApi —— 会导致界面看不到）。
