# Cherry Studio 聊天模块 — 模块说明

> 与 [源码地图.md](./源码地图.md) 配套。本文档逐模块描述职责、依赖与对外接口。

## 1. UI 层

### 1.1 HomePage（`pages/home/HomePage.tsx`）
**职责：** 整屏入口；持有 `activeAssistant` / `activeTopic` 两个本地状态；挂载 Navbar、HomeTabs、Chat。
**依赖：** `useAssistants()`、`useActiveTopic()`、`useShortcut()`、`@tanstack/react-router`。
**对外接口：** 通过 props 把 `setActiveTopic` 下传给 `Tabs`。

### 1.2 Chat（`pages/home/Chat.tsx`）
**职责：** 三栏布局协调；包裹 `QuickPanelProvider`；条件渲染右侧 Topics 抽屉。
**依赖：** Messages、Inputbar、ChatNavbar、`useChatContext()`。
**UI 栈：** 仍用 `antd.Flex` + styled-components，待迁 `@cherrystudio/ui` 的 `RowFlex/ColFlex`。

### 1.3 Tabs/Topics（`pages/home/Tabs/components/Topics.tsx`）
**职责：** 左侧 Topic 列表；CRUD（新建 / 重命名 / 删除）；拖拽排序；右键菜单；批量管理模式。
**数据源：** `useAssistant(assistantId)` 返回的 `assistant.topics[]`（Redux），加上 `useCache('topic.renaming', ...)` 做临时态。
**UI 栈：** 混合 — `@cherrystudio/ui` 的 `ContextMenu` / `Button`，但仍依赖 `antd.Tooltip` / `antd.Dropdown`。

### 1.4 Messages（`pages/home/Messages/Messages.tsx`）
**职责：** 消息流容器；倒序虚拟无限滚动；按 `parentId / askId` 把同问题多回复分组；监听事件总线 `SEND_MESSAGE` / `CLEAR_MESSAGES` / `EDIT_CODE_BLOCK` / `NEW_BRANCH`。
**数据源：** `useTopicMessages(topicId)` → `selectMessagesForTopic`。

### 1.5 Message / MessageContent / Blocks/index.tsx
**职责链：**
- `Message.tsx`：单条消息壳（头部、菜单、编辑器）。
- `MessageContent.tsx`：mentions 展示 + 调用 `MessageBlockRenderer`。
- `Blocks/index.tsx`：**核心分发器**。从 Redux 取出 `messageBlocksSelectors.selectEntities`，按 `block.type` 路由到对应子组件；对 IMAGE/TOOL 等连续同型块做分组合并；包裹 `AnimatedBlockWrapper` 做 motion 过渡；`ErrorBoundary` 兜底。

### 1.6 Markdown（`pages/home/Markdown/Markdown.tsx`）
**职责：** ReactMarkdown + remark / rehype 插件链；自定义 `CodeBlock`、`Link`、`Table`、KaTeX、SVG 渲染；**通过 `useSmoothStream` 实现打字机效果**。
**流式机制：** 监听 `block.content` 变更 → 计算 delta → `addChunk(delta)`；当 `block.status !== 'streaming'` 停止动画。

### 1.7 Inputbar（`pages/home/Inputbar/Inputbar.tsx`）
**职责：** 输入框；附件上传；@模型 mentions；知识库选择；token 估算；发送。
**发送流程：**
1. `FileManager.uploadFiles(files)`
2. `getUserMessage({ assistant, topic, content, files, mentions })`
3. `dispatch(_sendMessage(message, blocks, assistant, topicId))`
4. 清空输入

## 2. 状态层（Redux，v1）

### 2.1 store/index.ts
- `persist` key = `'cherry-studio'`，version = 206，storage = `localStorage`
- **blacklist**：`runtime` / `messages` / `messageBlocks` / `tabs` / `toolPermissions`（即聊天数据 **不**走 redux-persist，靠 Dexie 持久 + 启动时 hydrate）

### 2.2 newMessage slice（`store/newMessage.ts`）
State：
```ts
{
  entities: Record<id, Message>          // EntityAdapter
  ids: string[]
  messageIdsByTopic: Record<topicId, id[]>
  currentTopicId: string | null
  loadingByTopic, fulfilledByTopic       // 加载态
  displayCount: number
}
```
关键 actions：`setCurrentTopicId` / `messagesReceived` / `addMessage` / `insertMessageAtIndex` / `updateMessage` / `removeMessage` / `removeMessagesByAskId` / `clearTopicMessages` / `upsertBlockReference` / `setTopicLoading`。
关键 selector：`selectMessagesForTopic(state, topicId)`。

### 2.3 messageBlock slice（`store/messageBlock.ts`）— **@deprecated v2.0.0**
State：
```ts
{
  entities: Record<id, MessageBlock>
  ids: string[]
  loadingState: 'idle'|'loading'|'succeeded'|'failed'
  error: string | null
}
```
关键 actions：`upsertOneBlock` / `upsertManyBlocks` / `removeOneBlock` / `updateOneBlock`。
关键 selector：`messageBlocksSelectors.selectById`、`selectFormattedCitationsByBlockId`、`selectActiveTodoInfo`。

### 2.4 assistants slice（`store/assistants.ts`）
State 含 `defaultAssistant`、`assistants[]`、`presets[]`、`tagsOrder`、`unifiedListOrder`。每个 `assistant.topics[]` 直接挂在助手对象内。

### 2.5 messageThunk（`store/thunk/messageThunk.ts`）— **@deprecated v2.0.0**
关键 thunk：`sendMessage`、`appendAssistantResponseThunk`、`regenerateAssistantResponseThunk`、`resendMessageThunk`、`resendUserMessageWithEditThunk`、`updateMessageAndBlocksThunk`、`removeBlocksThunk`、`clearTopicMessagesThunk`、`loadTopicMessagesThunk`、`deleteSingleMessageThunk`、`deleteMessageGroupThunk`、`updateTranslationBlockThunk`。
内含 TODO：`// TODO: Migrate block deletion to Data API when block endpoints are available`。

### 2.6 migrate.ts
redux-persist `migrate` 函数表，按 version 字符串映射；最新 version 206。每个版本一段升级逻辑，常见操作：addProvider、UI 设置增删、字段重命名。

## 3. 持久化层

### 3.1 Dexie / IndexedDB（v1，**@deprecated**）
- 入口：`src/renderer/src/databases/index.ts`（顶部注明 *Scheduled for removal in v2.0.0*）
- 升级历史：`upgrades.ts`，v1→v10
- 表：`files` / `topics`（messages[] 内联）/ `message_blocks` / `settings` / `translate_history` / `quick_phrases` / `knowledge_notes` / `translate_languages`
- v7 升级关键：把旧 `message.content` 单字段拆为多个 `MessageBlock` 行
- 当前用法：聊天数据「真源」仍是 Dexie，Redux 仅作内存缓存；启动通过 `loadTopicMessagesThunk` 从 Dexie hydrate

### 3.2 SQLite + Drizzle（v2，目标态）
- 入口：`src/main/data/db/`
- 关键 schema：`message.ts` / `topic.ts` / `assistant.ts` / `userModel.ts` / `file.ts`
- 反规范化：v2 `message.data.blocks` 内联 JSON，不再独立 `message_blocks` 表
- FTS5：`message_fts` 虚拟表 + 触发器自动同步（trigram 分词）
- 树形：`message.parentId` + `siblingsGroupId` 替代 v1 的 `askId`
- 访问层：renderer 通过 `useQuery('/topics')` / `useMutation('/messages', 'POST')`（`src/renderer/src/data/hooks/useDataApi.ts`） — 聊天 UI **尚未消费**

## 4. AI 调用层

### 4.1 流程
```
Inputbar.sendMessage
  → messageThunk.sendMessage
    → streamingService.createUserMessage / createAssistantMessage
    → getTopicQueue(topicId).add(fetchAndProcessAssistantResponseImpl)
      → ApiService.transformMessagesAndFetch
        → ConversationService.prepareMessagesForModel
        → fetchChatCompletion
          → AiProvider.completions
            → PluginBuilder.buildPlugins
            → createExecutor (packages/aiCore)
            → executor.streamText (Vercel AI SDK)
              → HTTP → LLM Provider（renderer 直连）
              → ReadableStream
            → AiSdkToChunkAdapter.processStream → Cherry Chunk
            → onChunkReceived
              → StreamProcessingService.createStreamProcessor
                → callbacks/{text,tool,image}Callbacks.*
                  → BlockManager.smartBlockUpdate
                    → StreamingService.updateBlock
                      → Redux upsertOneBlock (throttled)
                      → 完成时 finalize → DataApi 落库
```

### 4.2 AiProvider（`aiCore/AiProvider.ts`）
- `completions()` 主入口
- 用 `createExecutor` 构建执行器
- `modernCompletions()` 处理流式响应

### 4.3 插件 / 中间件
- `aiCore/plugins/PluginBuilder.buildPlugins()` 动态装配
- 插件名义：reasoning、web-search、function-calling、tool-use
- 中间件配置类型：`aiCore/types/middlewareConfig.ts` → `AiSdkMiddlewareConfig`
- 底层基于 **Vercel AI SDK**；`AiSdkToChunkAdapter` 把 SDK chunk 转成 Cherry 的 `Chunk`

### 4.4 ChunkType（`types/chunk.ts`）
`TEXT_DELTA` / `TEXT_COMPLETE` / `THINKING_DELTA` / `BLOCK_COMPLETE` / `MCP_TOOL_*` / `ERROR`。

### 4.5 队列（`utils/queue.ts`）
- `getTopicQueue(topicId)` — 一个 topic 一个 queue；保证同一会话内消息按序处理
- `waitForTopicQueue(topicId)` — 等待队列空
- 作用：防止用户连发时多个回复并行 race

### 4.6 IPC 边界
**LLM 请求由 renderer 直连第三方 API**（OpenAI / Anthropic / 自托管…）。Main 进程不参与 LLM 流。Main 端 IPC 主要承担：
- 知识库 RAG (`IpcChannel.Knowledge_*`)
- 文件 (`IpcChannel.File_*`)
- 代理会话历史 (`IpcChannel.AgentMessage_GetHistory`)
- DataApi 业务数据（v2 走 `useQuery`/`useMutation`）

## 5. Hook 层

| Hook | 文件 | 作用 |
|---|---|---|
| `useTopicMessages` | `hooks/useMessageOperations.ts` | 读 topic 下所有消息 |
| `useMessageOperations` | 同上 | 删 / 编辑 / 重发 / 重新生成 |
| `useChatContext` | hooks 内 | 当前会话上下文 |
| `useAssistant` | `hooks/useAssistant.ts` | 单个助手（含 topics） |
| `useAssistants` | 同上 | 助手列表 |
| `useActiveTopic` | hooks 内 | 当前激活 topic |
| `useSmoothStream` | `hooks/useSmoothStream.ts` | 打字机动画 |
| `usePreference` | data 层 | v2 设置读写（已可用） |
| `useCache` / `useSharedCache` | data 层 | 临时态（如 renaming） |
| `useQuery` / `useMutation` | `data/hooks/useDataApi.ts` | v2 业务数据访问（聊天侧未启用） |

## 6. 共享包

### 6.1 `packages/aiCore`
通用 AI 执行框架。导出 `createExecutor` / `streamText` / `generateImage` / 插件系统。底层封装 Vercel AI SDK。

### 6.2 `packages/ai-sdk-provider`
Cherry 自定义 Provider（`cherryin-provider.ts`），承担 Cherry 特有的 provider 适配 + 中间件。

### 6.3 `packages/shared/data/types`
v2 类型权威定义：`message.ts`、`topic.ts`、`assistant.ts`（含 zod schema）。renderer 与 main 共用。

### 6.4 `packages/ui`（`@cherrystudio/ui`）
Shadcn + Tailwind 组件库。聊天 UI 已开始用 `Avatar`、`RowFlex/ColFlex`、`Tooltip`、`ContextMenu`、`Button`，但覆盖率低。

## 7. 主进程聊天相关 Service

> 用 lifecycle 容器管理（`@Injectable` + `@ServicePhase`）。具体清单见 `src/main/core/application/serviceRegistry.ts`。

聊天相关：
