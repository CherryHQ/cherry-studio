## 0. 一图速览

```
用户输入 (Inputbar)
    │
    ▼
Redux Thunk: messageThunk.sendMessage()
    ├─→ 持久化：StreamingService.createUserMessage() → DataApi/Dexie
    ├─→ Redux：newMessagesActions.addMessage()
    └─→ 队列：getTopicQueue(topicId).add(fetchAndProcessAssistantResponseImpl)
                │
                ▼
        ApiService.transformMessagesAndFetch
                │
                ▼
        fetchChatCompletion → AiProvider.completions
                │
                ▼
        packages/aiCore (createExecutor) → Vercel AI SDK executor.streamText
                │ HTTP → LLM Provider (renderer 直连，不走 IPC)
                ▼
        ReadableStream
                │
                ▼
        AiSdkToChunkAdapter.processStream → Cherry Chunk
                │
                ▼
        StreamProcessingService.createStreamProcessor (按 ChunkType 路由)
                │
                ▼
        messageStreaming/callbacks/* (textCallbacks / toolCallbacks / imageCallbacks …)
                │
                ▼
        BlockManager.smartBlockUpdate → StreamingService.updateBlock
                │
                ├─→ Redux: upsertOneBlock / updateOneBlock (throttled)
                └─→ 完成时 finalize → DataApi 落库
                │
                ▼
        UI: Messages → Message → MessageContent → Blocks/index.tsx
                │
                ▼
        各 Block 子组件 (MainTextBlock / ImageBlock / ToolBlock …)
                │
                ▼
        Markdown.tsx + useSmoothStream (打字机动画)
```

## 1. 顶层目录

| 路径 | 角色 |
|---|---|
| `src/main/` | Node 主进程：lifecycle 容器、SQLite DataApi、API Server、Knowledge、MCP、迁移器 |
| `src/preload/` | contextBridge 桥（IPC 暴露面） |
| `src/renderer/src/` | React 19 渲染层（聊天 UI 主战场） |
| `packages/aiCore/` | 通用 AI 执行框架（基于 Vercel AI SDK） |
| `packages/ai-sdk-provider/` | Cherry 自定义 Provider 实现 |
| `packages/shared/data/types/` | v2 数据类型定义（Message / Topic / Assistant / MessageDataBlock） |
| `packages/ui/` | `@cherrystudio/ui`：Shadcn + Tailwind 组件库 |
| `v2-refactor-temp/` | v2 重构沙箱：classification.json、迁移文档、breaking changes |

## 6. v1 / v2 共存现状一句话（2026-05-20 修正）

**数据层：** v1 Dexie/Redux 标 `@deprecated`，v2 SQLite + Drizzle schema 完整，迁移器 (`ChatMigrator`) 已能跑通。
**业务层（写）：** 已部分走 DataApi —— `POST /topics`、`POST /topics/:topicId/messages`、`PATCH /messages/:id` 在用；流式 in-flight 状态在 `cacheService`，finalize 时一次性 PATCH 落库。
**业务层（读）：** 通过 `DataApiMessageDataSource`（自标"临时兼容层"）从 DataApi 拉数据 → 转 v1 类型 → 喂 Redux → UI 订阅 Redux。准确说是 **「DataApi 是真源、Redux 是显示缓存」**。
**UI 层：** 聊天区仍以 antd + styled-components 为主；少数（MessageHeader / MessageAnchorLine 等）迁到 `@cherrystudio/ui`。
**分支对话（v2）：** schema + service + 测试 ✅；renderer UI ❌ 未启动（仍 v1 `askId` + `foldSelected`），等 `DeJeune/ai-service` 分支带 v2 `SiblingNavigator` / `MessageGroup` 合入。
