## 关键产出（节选）

**13 个端点全数有 schema + handler + service：**
- `/topics` GET/POST + `/topics/:id` GET/PATCH/DELETE + `/topics/:id/active-node` PUT + reorder
- `/topics/:topicId/tree` GET + `/topics/:topicId/messages` GET/POST + `/messages/:id` GET/PATCH/DELETE
- `/assistants` GET/POST + `/assistants/:id` GET/PATCH/DELETE
- `/temporary/topics` 全套 5 个

**真实分工**：写路径（`POST /topics`、`POST /topics/:topicId/messages`、`PATCH /messages/:id`）已落地；读路径走桥层 `DataApiMessageDataSource` 转 v1 类型喂 Redux；UI 订阅 Redux。**准确表述：DataApi 是真源、Redux 是显示缓存**。

**缺口**：
1. 无块级端点（块 ops 走 message data 整体替换）
2. Agent session 写路径仍走 dbService
3. `DataApiMessageDataSource` 桥层待删
4. `POST /topics/:topicId/messages` 不传 parentId 时多窗口竞态
