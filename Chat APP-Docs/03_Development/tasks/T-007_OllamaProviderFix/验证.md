### 期望现象（4 项验收目标）

| # | 检查点 | 期望 | 失败信号 |
|---|---|---|---|
| 1 | 进 Ollama Provider 设置页，看 API 字段的**标题** | `API Host` | 如显示 `Anthropic API Host` → 修复未生效 |
| 2 | 看 API 字段下方的 **preview URL** | `http://localhost:11434/api/chat` | 如显示 `/v1/messages` 或 `/v1/chat/completions` → primaryEndpoint 解析没短路到 ollama-chat |
| 3 | 看 Provider 顶部的 **Model list 计数** | `N/N Enabled`（N = 本机 ollama 模型数） | 如仍 `0/0` → 看 DevTools console 是否有 `Provider auto model sync failed` |
| 4 | 顶栏 Select Model 弹窗搜 `gemma` 或 `qwen2.5:7b-instruct` 或你本机有的任意模型名 | 命中并能选中 | 搜不到 → 同 #3 |

### 不期望出现

- ❌ `Provider auto model sync failed`（应是 'ready' → 'Completed'）
- ❌ `Invalid JSON response`
- ❌ Ollama 设置页任何 Anthropic 标签 / `/v1/messages` 字样

### 如果验证失败的排查顺序

1. 确认本地 `packages/provider-registry/data/providers.json:425` 确实是新行（`git diff` 应该只有这一行 +）
2. 确认是 fresh install（`ls ~/Library/Application\ Support/CherryStudioDev` 不存在）—— 如果有老 DB，`user_provider` 表的 ollama 行 `defaultChatEndpoint` 仍是 NULL，修复不会生效
3. DevTools → Application → IndexedDB / Local Storage 看是否还有遗留 v1 state
4. 如确认 fresh install 但仍 fail：把 console 完整 log + 设置页截图反馈，可能是 Ollama SDK 版本/适配的另一类问题
