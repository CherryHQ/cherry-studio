## 手动验证（可选，等 T-006C 联动）

T-006B 单独不暴露 UI 变化（只是 DOM 上多了两个 data 属性 + 一层透明 div）。但是可以快速 sanity check：

### 推荐做法 A：打开 DevTools 直接看

```bash
pnpm dev
# 等聊天界面渲染出至少一条 assistant message
# DevTools → Elements → 选中任意 assistant message 的文本区
# 应该看到结构：
#   <div class="message-content-container">
#     ...
#     <div data-message-id="<uuid>" data-block-id="<uuid>">  ← T-006B 加的
#       <div class="markdown">
#         <p>...</p>
#       </div>
#     </div>
#   </div>
```

### 推荐做法 B：在 DevTools Console 跑一次 helper

```js
// 选中任意一段 assistant 文本，然后在 Console 跑：
const range = window.getSelection().getRangeAt(0)
// 临时手动 import 不可行，但可以复制 helper 逻辑：
const startBlock = (range.startContainer.nodeType === 1 ? range.startContainer : range.startContainer.parentElement).closest('[data-block-id]')
const messageEl = startBlock?.closest('[data-message-id]')
console.log({
  blockId: startBlock?.getAttribute('data-block-id'),
  messageId: messageEl?.getAttribute('data-message-id')
})
```
应输出 UUID。

## 已知限制

- 不会自动检测「选区跨 message」 —— 但实际上跨 message 选区时 `startContainer` 和 `endContainer` 解出不同 blockId（即使是不同 messages 的 blocks），cross-block 判断也会返回 null，所以行为正确（disabled）。仅是：诊断信息不区分「跨 block 同 message」与「跨 message」。可接受。
- 流式期间渲染的 block：data 属性正确，但 helper 不感知 streaming state（T-006C 负责挂 disable）
