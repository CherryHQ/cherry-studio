## 顺序建议

1. **T-006B 先做**：基础设施（DOM data 属性 + climb helper）—— 独立可验证（vitest 跑得通）
2. **T-006A 跟上**：有 testbed 后做 fixture 更顺手
3. **T-006C**：把 selection 接到菜单
4. **T-006D**：Panel UI + cache + fork 调用
5. **T-006E**：高亮叠在 C/D 完成之后
6. **T-006F**：评估用户反馈后决定

## 跨子任务的工程纪律备注

按 CLAUDE.md：
- **`@cherrystudio/ui`** 必用（ContextMenu / Sheet / Drawer / Button / Input 都从这里取）—— 不引入 antd、styled-components
- **i18next**：所有新 UI 字符串走 i18n，跑 `pnpm i18n:check` 验证
- **logger**：新代码用 `loggerService.withContext('BranchPanel' / 'BranchAnchor')`，禁止 console.log
- **测试**：T-006B 至少补 1 个 vitest（DOM climb helper）；UI 组件按情况补；CLAUDE.md「无测试 = 未完成」
- **commit**：Conventional + `--signoff`；scope 用 `feat(branch-ui): ...` / `feat(messages): ...`
