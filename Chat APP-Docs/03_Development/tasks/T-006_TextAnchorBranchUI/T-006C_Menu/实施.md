## 1. SelectionContextMenu 改动要点

### 1.1 新 import
```ts
import { type BlockContext, findBlockContext } from '@renderer/utils/branchAnchor/findBlockContext'
```

### 1.2 新状态
```ts
const [blockContext, setBlockContext] = useState<BlockContext | null>(null)
```

### 1.3 `handleOpenChange` 扩展
旧逻辑：只 setSelectedText
新逻辑：额外调 `findBlockContext(selection.getRangeAt(0))` 并 setBlockContext
无选区分支：两个 state 都清空

### 1.4 两个新 handler（占位）
```ts
const handleAskHere = () => {
  if (!blockContext) return
  logger.debug('branch-anchor: ask here', {
    messageId: blockContext.messageId,
    blockId: blockContext.blockId,
    selectedText
  })
}

const handleOpenAsBranch = () => {
  if (!blockContext) return
  logger.debug('branch-anchor: open as branch', { ... })
}
```

按 CLAUDE.md「Never use console.log — always use loggerService」用 `logger.debug`（既有 `loggerService.withContext('SelectionContextMenu')` 已在文件顶部声明）。loggerService 底层走 console.debug 输出，符合用户「console.debug 中输出」的语义。

### 1.5 两个新菜单项
```tsx
const hasAnchor = hasSelection && blockContext !== null

<ContextMenuItem disabled={!hasAnchor} onSelect={handleAskHere}>
  {t('chat.message.anchor.ask_here')}
</ContextMenuItem>
<ContextMenuItem disabled={!hasAnchor} onSelect={handleOpenAsBranch}>
  {t('chat.message.anchor.open_as_branch')}
</ContextMenuItem>
```

排在 Copy / Quote 之后；菜单顺序 = Copy → Quote → Ask Here → Open As Branch。

### 1.6 启用 / 禁用矩阵

| 场景 | Copy | Quote | Ask Here | Open As Branch |
|---|---|---|---|---|
| 单 assistant block 内选中文字 | ✅ | ✅ | ✅ | ✅ |
| 单 user block 内选中文字 | ✅ | ✅ | ❌ | ❌ |
| 跨 block 选区 | ✅ | ✅ | ❌ | ❌ |
| 无 data 属性区域选区（error card / 散落文本） | ✅ | ✅ | ❌ | ❌ |
| 无选区 / 选区 collapsed | ❌ | ❌ | ❌ | ❌ |

**关键**：Ask Here / Open As Branch 多两个门槛 —— `blockContext !== null` **且** `blockContext.role === 'assistant'`；Copy / Quote 行为完全不变。

### 1.7 Role 信息来源（事后补丁）

`MainTextBlock` 的 wrapper `<div>` 在 `data-message-id` / `data-block-id` 之上再带 `data-message-role={role}`（值同 `Message['role']`）。`findBlockContext` 把它读出来，挂到 `BlockContext.role` 上（合法值 `'user' | 'assistant' | 'system' | null`，未知值降级为 `null`）。`SelectionContextMenu` 只在 `role === 'assistant'` 时点亮 Ask Here / Open As Branch ——

- user 消息没有「从这里分叉」的语义（产品决策：分支总是从 AI 回答展开）
- error card / Citations 列表 / 任何不走 `MainTextBlock` 的区域 → 没有 `data-message-role` → `role = null` → 默认不算 assistant，分支项保持 disabled

### 测试技巧

- **本地 override 全局 `@cherrystudio/ui` mock**：全局 mock 不连 `onOpenChange`，本地版本用 Context 把 onOpenChange 通过 ContextMenuTrigger 的 `onContextMenu` 触发，让真实的 `handleOpenChange` 跑起来
- **`react-i18next` mock 返回 key 字符串本身**：assert 时直接用 key 名找菜单项，避免维护 label 映射
- **`vi.spyOn(window, 'getSelection')`**：每个测试构建专属 Range + Selection；不污染全局
- **`vi.spyOn(console, 'debug')`**：拦截 loggerService 委派的 console 输出（全局 RendererLoggerService mock 用 `console.debug` 实现）
- **eslint-disable-next-line** 在 mock 内对 `Children.only` + `cloneElement` —— 这两个 React 19 idiom warning 是模拟 Radix `asChild` 的必需操作
