## 设计决策（实施时）

| 选择 | 理由 |
|---|---|
| `logger.debug` 而非 `console.debug` | CLAUDE.md「禁用 console.log」；底层 RendererLoggerService 走 console.debug，效果等价 |
| 菜单项放 Copy/Quote **之后** | 文本动作 → 文本动作 → 上下文动作的自然分层 |
| **不**加 `data-block-streaming` | 严守 Surgical Changes；T-006C 设计原本带，但用户「不做超出 T-006C 范围的重构」明确要求 |
| **不**加 cross-block hint tooltip | ContextMenuItem 无原生 tooltip 支持；不在用户需求里；disabled 状态本身已给视觉信号 |
| i18n key 用 `chat.message.anchor.*` 命名空间 | 与既有 `chat.message.new`、`chat.message.quote` 同层；alphabetical 排序后自然在前 |
| 本地 override `@cherrystudio/ui` mock 而不是改全局 setup | 全局 mock 不连 `onOpenChange`，全局改影响面太大；测试本地需要的特定行为 |

## 经验

1. **全局测试 setup 是宝藏也是陷阱** —— `tests/renderer.setup.ts` 已经 mock 了 `@cherrystudio/ui` 大部分组件，但 ContextMenu 的 `onOpenChange` 没连。要么改全局（影响 6+ 测试文件），要么本地 override —— 选本地，scope 干净
2. **ESLint 警告 vs 错误**：`pnpm lint` 跑 eslint **不带** `--deny-warnings`（只 oxlint 带），但保持 0 warning 仍是好习惯。React 19 idiom 警告里前 3 条好治（`Context` 命名、`<Context>` 用法、`React.use`），后 2 条（`Children.only` + `cloneElement`）是模拟 `asChild` 的必需操作，用 `eslint-disable-next-line` 局部抑制 + 注释解释最妥
3. **i18n alphabetical insertion** 一次就过 —— `chat.message.anchor` 字典序在 `new` 之前；手插对位置 + `pnpm i18n:check` 直接通过，省了 `pnpm i18n:sync` 一轮

## 给后续任务的接口

### T-006D 直接可用

T-006C 的两个 handler 现在是 logger.debug 占位。T-006D 需要：

```ts
// 在 SelectionContextMenu.tsx 替换 handleAskHere / handleOpenAsBranch
import { useBranchAnchors } from '@renderer/hooks/useBranchAnchors'

const { add, openPanel } = useBranchAnchors()

const handleOpenAsBranch = () => {
  if (!blockContext) return
  const anchorId = crypto.randomUUID()  // 或用 uuid v7
  add({ anchorId, messageId: blockContext.messageId, blockId: blockContext.blockId, selectedText, ... })
  openPanel(anchorId, 'fork')
}

const handleAskHere = () => {
  if (!blockContext) return
  const anchorId = crypto.randomUUID()
  add({ anchorId, ... })
  openPanel(anchorId, 'inline-ref')
}
```

并把 `range` 一起传给 T-006E 的 `highlightSelection(range, anchorId)`。

注意：当前 handler 内只持有 `selectedText` + `blockContext`，**没有 range 对象**。T-006D 实施时需要在 `handleOpenChange` 把 `range`（或它的克隆 / 起止 offset）也存 state，供 handler 使用。
