## 手动验证（推荐执行）

T-006C 起开始有 UI 变化。建议跑：

```bash
rm -rf ~/Library/Application\ Support/CherryStudioDev   # 可选：fresh install
pnpm dev
```

### 期望现象

1. 启动 → 新建 topic → 任意发一条消息（用任意非 Ollama provider；D-003 已知）
2. 在 AI 回复的某段文字上**选中**，右键
3. 看到 4 个菜单项：
   - Copy
   - Quote
   - **Ask about this**（针对此处提问）
   - **Open as branch**（展开为分支）
4. 点击 "Ask about this" 或 "Open as branch"
5. 打开 DevTools → Console，看到 logger.debug 输出：
   ```
   [SelectionContextMenu] branch-anchor: ask here { messageId: '<uuid>', blockId: '<uuid>', selectedText: '<选中的文字>' }
   ```
   或
   ```
   [SelectionContextMenu] branch-anchor: open as branch { ... }
   ```

### Disabled 现象

- 不选任何文字直接右键 → 4 个菜单项都 disabled
- 选区跨多段（如跨两个 `<p>` 跨 block）→ "Ask about this" / "Open as branch" disabled，Copy / Quote 仍可用
- 选区落在**自己刚发的 user 消息**上 → "Ask about this" / "Open as branch" disabled，Copy / Quote 仍可用
- 选区落在 error card / Citations 列表等没有 `data-message-role` 的区域 → "Ask about this" / "Open as branch" disabled，Copy / Quote 仍可用
- 在 AI 回复区域之外（如标题栏、设置页）选中文字 → 由于不在 `SelectionContextMenu` 包裹范围内，**菜单不会出现**

### 不期望的现象

- ❌ Copy / Quote 行为改变（点击之后没复制 / 没引用）
- ❌ 任何 console.error
- ❌ 菜单显示 i18n key 字符串（如显示 `chat.message.anchor.ask_here` 字面量），如果出现 = i18n 加载有问题
