## 设计要点

### 1. State 放在哪？

候选：(a) Context Provider；(b) useCache；(c) Redux；(d) module-level store。
**选 (a) 的轻量版 = `Messages.tsx` 本地 useState** —— 最小、无新依赖、易测。D-2 切到 `useCache` 时只换 setBranchAnchor 的实现细节即可。

### 2. UI 容器选 Dialog 还是 Drawer？

`@cherrystudio/ui` 都导出。**选 Dialog**：
- 全局测试 mock 已覆盖 Dialog 全套（DialogContent/Header/Footer/Title），Drawer 没 mock；改全局影响面大
- 「壳子原型」用居中 Dialog 也能完成验收；Drawer（侧滑）是 v1 体验优化，可放到 D-3
- CLAUDE.md「Simplicity First」+「No abstractions for single-use code」

### 3. SelectionContextMenu 的 callback 而非 Context

`SelectionContextMenu` 是通用组件（也被 `CitationsList.tsx` 用，T-006C 调查所得）。**不能强加 Context 依赖**，否则非 host 实例会 crash。
做法：`onOpenBranchPanel?: (anchor) => void` 可选 prop —— 有 callback 时调，没有时回到 T-006C 的 `logger.debug` 兜底。原有测试不动，新加 1 个 callback 优先级用例。

### 4. BranchAnchor 类型放在 BranchPanel/

`findBlockContext` 已经 export `BlockContext = { messageId, blockId }`。`BranchAnchor = BlockContext + selectedText`。
**新文件 `BranchPanel/types.ts`** 而不是塞进 `findBlockContext.ts` —— Anchor 是 Panel 的领域类型；BlockContext 是 DOM 解析的中间态。两者解耦更清晰。

### 5. 追问输入框的 state

- BranchPanel 内部 `useState('')`
- `useEffect` 监听 `open` 变化，在 close 时清空
- 不外提到 props，保持组件自治
- 测试覆盖了「关闭再开 → 输入清空」

### 6. 创建按钮：disabled 还是 log？

用户原话「创建分支按钮先 disabled 或只打日志」—— 选 **log + 关闭 panel**：
- 触发 logger.debug 输出捕获的完整 anchor + followUp（便于手测确认数据完整）
- 关闭 Panel（避免 zombie 状态）
- 在 `anchor === null` 防御场景下才 disabled
