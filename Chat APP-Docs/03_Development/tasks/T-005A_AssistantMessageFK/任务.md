## 结论
**FK 失败字段 = `modelId`** 指向 `user_model(id)`。
**根因**：`messageThunk.ts:802, 1062` 等 5 处 createAssistantMessage 调用点把 v1 `Model.id`（短字符串）当 v2 `UniqueModelId`（`"providerId::modelId"` 格式）传过去；fresh install 后 user_model 表为空，更无可能有 id=`'qwen'` 的行。
**推荐修复**：在 `StreamingService.createAssistantMessage`（一处）的 DTO 构造里集中清洗：`isUniqueModelId(modelId) ? modelId : undefined`。
