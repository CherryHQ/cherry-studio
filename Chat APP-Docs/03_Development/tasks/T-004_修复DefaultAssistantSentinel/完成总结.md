## 经验

1. **CLAUDE.md「v1 是 throwaway」不等于「v1 是 deadcode」** —— v1 代码（这里是 `mapLegacyTopicToDto`）在 v2 完整切换前仍是活路径，需要做最小适配让两边契约对齐
2. **「为什么 rm 不行」是好诊断切入点** —— 把"清数据是否解决问题"作为分类标准，能立刻区分「数据问题 vs 代码问题」
3. **跨 v1/v2 边界的 sentinel 是高危点** —— 同源问题大概率不止一个：`useAssistantPresets` 已经在等 fix；其它 `'default'` 出现在 8 处测试文件 + AssistantService.ts:79 本体，需要更大范围的 sentinel-elimination 工程
4. **修复时机选择**：方案 B 一行修动作小、风险低，但**不彻底**；CLAUDE.md「Simplicity First」+「Fix upstream, don't hack downstream」要求：长期 fix 应该在 `getDefaultAssistant()` 上消除 sentinel（方案 C 路线），而非在 `mapLegacyTopicToDto` 这种 downstream 加判断。本任务选 B 是 **baseline unblock** 的临时取舍，不是终局
