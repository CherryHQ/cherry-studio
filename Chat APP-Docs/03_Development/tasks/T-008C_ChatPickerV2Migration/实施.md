## 1. chat-model-popup.tsx 改动要点

### 1.1 数据源切换

**Before**：
```ts
import { useAllProviders, useProviders } from '@renderer/hooks/useProvider'
// ...
const { providers } = useProviders()                    // v1 Redux: enabled + concat CHERRYAI
const allProviders = useAllProviders()                  // v1 Redux: 全表（含 disabled）
```

**After**：
```ts
import { useModels } from '@renderer/hooks/useModels'
import { useProviders as useV2Providers } from '@renderer/hooks/useProviders'
import { toV1ModelShim, toV1ProviderShim }
  from '@renderer/pages/settings/ProviderSettings/utils/v1ProviderShim'
// ...
const { providers: v2Providers } = useV2Providers({ enabled: true })   // v2 DataApi
const { models: v2Models } = useModels({ enabled: true })              // v2 DataApi
```

### 1.2 合并逻辑（v2 → v1）

- `groupBy(v2Models, 'providerId')` 按 providerId 桶分
- 每个 v2 Provider → `toV1ProviderShim(...)` 转 v1-shape；带的 models = 该 providerId 的 v2 Models → `toV1ModelShim(...)` 转 v1-shape
- `m.isHidden` 模型直接排除（`user_provider.isEnabled` 已在 hook 入口的 `enabled:true` 处过滤）
- caller 的 `filter` 函数在 v1 shape 上执行，与改前完全一致
- 空 models 的 provider 直接丢弃（与改前同）

### 1.3 CHERRYAI fallback（option A）

```ts
const hasCherryai = fromV2.some((p) => p.id === 'cherryai' && p.models.length > 0)
if (hasCherryai) return fromV2

const cherryaiModels = filter ? CHERRYAI_PROVIDER.models.filter(filter) : CHERRYAI_PROVIDER.models
if (cherryaiModels.length === 0) return fromV2
return [...fromV2, { ...CHERRYAI_PROVIDER, models: cherryaiModels }]
```

- 当 v2 user_model 已经有真实 cherryai 行（如用户登录 CherryIN 后 sync 了模型）→ 不注入，避免重复
- 当没有 → 把硬编码 `CHERRYAI_PROVIDER`（含 demo `qwenModel`）追加在末尾
- filter 也作用于 fallback 模型（避免在 embedding filter 下混入聊天模型）

### 1.4 PopupContainer 改为 named export

为了让 unit test 能直接 mount 而不走 TopView 包装：

```ts
// Before: const PopupContainer: React.FC<Props> = ...
export const PopupContainer: React.FC<Props> = ...
```

JSDoc 注明这是测试 affordance，生产仍走 `SelectChatModelPopup.show(...)`。
