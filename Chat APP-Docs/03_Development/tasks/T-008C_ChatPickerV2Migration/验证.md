### 如果验证失败的排查顺序

1. **picker 里看不到 Ollama**：
   - 先确认 Provider 设置页能看到（如果不能 → D-003B 没生效，回去检查）
   - DevTools Console: `await window.api.dataApi.query('/models', { query: {} })` 应该返回 Ollama 模型行；如果空 → v2 sync 没成功
   - 把 `chat-model-popup.tsx` 里 `const filteredProviders = useMemo(...)` 包一层 `console.log` 打印 v2Providers / v2Models 看到底从 hook 拿到什么
2. **能看到但选不中 / 选中报错**：
   - 检查 caller（`SelectModelButton.tsx:36`）的 `updateAssistant({ model: selectedModel })` —— assistant.model 应该是 v1 shape；如果不是，shim 转换有问题
3. **选中后发消息 FK 报错**：
   - 看 console 报错的 modelId 值；如果是 `ollama::qwen2.5:7b-instruct` → 说明 shim 没把 id 转成 raw，v1ProviderShim.ts 有 bug
   - 如果是 null → 跟今天行为一致，FK 不该报；可能是其它问题
