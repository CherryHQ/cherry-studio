### 任务

### 结果概要
- **Mutation 1**（paint = no-op）：sourceHighlight 6 个 paint/clear 用例里的 5 个 RED + MainTextBlock 5 个 highlight 用例里的 3 个 RED。9 个 round-trip + 3 个「不应注入」用例 GREEN，符合预期。
- **Mutation 2**（clear = no-op）：sourceHighlight 3 个 RED（clear / A→B / 循环），符合预期；其余 GREEN。但 **MainTextBlock 那条 "removes all injected spans when the component unmounts (effect cleanup)" 也 GREEN —— 这是个 fake-green**。
- **Fake-green 根因**：用 testing-library 的 `unmount()` 会把整个 React 子树从 DOM 容器中拆掉；painted span 在 wrapper `<div>` 内随子树一并消失，**不是**因为 effect cleanup 真的调了 `clearSourceHighlight`。test name 暗示的是 "effect cleanup"，实际只测了 "unmount 后 DOM 没了"。形状跟原 `data-branch-anchored` 假绿一模一样。
- **修法**：1-for-1 替换为 deps-change cleanup 用例 —— 用 `rerender` 把 BranchAnchorContext value 从匹配翻成不匹配，组件保持 mounted，spans 必须由 effect cleanup 调 `clearSourceHighlight` 才能消失。React 不再有"子树拆除"这个掩盖通道。
- **新用例对 Mutation 2 的反应**：RED ✓（清扫被破坏 → spans 不消失 → 第二个 `toHaveLength(0)` 断言失败）。覆盖目的达成。
- **revert 验证**：每轮 mutation 后 `git checkout -- sourceHighlight.ts`、`git diff` 为空、focused 测试 36/36 全绿。

- **临时会话端点不对称**：无单条读、无树、无 branch 分页 —— 设计上临时会话本就线性

### 关键发现（摘录）

- **会话载体就是 Topic**（v1、v2 都是），无独立 `Conversation` 类型。
- **聊天 UI 数据流是 v1（Redux + Dexie）100% 主导**：5+ 个核心聊天组件实测仅订阅 Redux，DataApi 端点已就位但聊天侧未消费。
- **Redux 聊天 slices 已标 `@deprecated v2.0.0`**：`store/messageBlock.ts` 顶部 + `store/thunk/messageThunk.ts` 顶部。`messageThunk.ts:1332` 留有 `// TODO: Migrate block deletion to Data API when block endpoints are available`。
- **流式链路精确切分**：`AiSdkToChunkAdapter`（SDK → Cherry Chunk）+ `StreamProcessingService.createStreamProcessor`（路由）+ `messageStreaming/callbacks/*`（处理）+ `BlockManager.smartBlockUpdate`（块更新）+ `StreamingService.updateBlock`（持久化 + Redux throttled dispatch）。
- **LLM 请求由 renderer 直连**，不走 IPC。Main 端只处理 RAG / 文件 / DataApi 业务数据 / Agent 历史。
- **v1 Block 与 v2 Block 的最大差**：v1 块独立表 + `message.blocks: string[]` 引用；v2 块直接内嵌 `message.data.blocks[]`（JSON）。
- **`CitationBlock` 在 v2 已废弃**，引用合并入 `MainTextBlock.references: ContentReference[]`。
- **redux-persist version 206**，blacklist 含 `messages / messageBlocks` — 即聊天数据不走 localStorage，依赖 Dexie 持久 + 启动时 hydrate。
