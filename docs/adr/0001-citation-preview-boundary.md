# Keep Citation Preview Separate From Web Search Providers

Citation Preview is presentation support for recognizing a referenced source, not the application's configured web fetch capability. We will move ordinary web citation preview fetching and parsing to the main process behind the narrow citation-specific IpcApi route `citation.fetch_preview`, which accepts one URL and returns only the cleaned short preview text. It will not reuse `web_search.fetch_urls`, live under a generic `chat` namespace, read the default web-search fetch provider, or accept a provider override. This keeps provider/API-key/quota/host failures from affecting citation panel rendering while still allowing the implementation to reuse lower-level safe remote-fetch primitives. Preview failures return an empty preview instead of surfacing UI errors; only IpcApi framework failures, such as validation or sender rejection, reject the renderer call.

Main-side preview work follows existing repository patterns: a small `PQueue` limits concurrent remote fetch/parse work, request timeouts start inside the queued task, and same-URL in-flight requests share one promise. Completed preview text stays in the renderer's existing SWR cache rather than gaining a separate main-process TTL cache.

The first migration is intentionally limited to ordinary non-X web citations rendered by `CitationsList`. `CitationTooltip` and X/Twitter oEmbed remain on their current renderer path. The citation route uses Electron `net.fetch`, validates the URL with `sanitizeRemoteUrl`, accepts only HTML/text responses, caps the response body at 1 MB, and applies an 8-second timeout after a queued task starts. The queue concurrency is three. There is no renderer-to-main cancellation protocol in this change.
