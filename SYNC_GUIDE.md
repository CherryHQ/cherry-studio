# Cherry Studio 专用多端同步指南

本版本的 Cherry Studio 深度集成了**零侵入的第三方多端同步功能**，能够让你将所有话题（Topics）在多台电脑间无缝同步。

## 运作原理
为了保证客户端极低的代码侵入性（避免与官方未来更新发生合并冲突），本功能**没有修改任何原版设置界面**，它通过静默读取本地的专属配置文件来无缝开启后台双向推送。

## 配合使用的同步服务器端
客户端的同步数据需要推送到一台服务器，你需要部署官方配套的极简 Node.js 脚本节点来接收和分发数据：
👉 **获取同步服务器代码：[ifastcc/cherry-sync-server](https://github.com/ifastcc/cherry-sync-server)**

## 如何在电脑上开启同步？ （分发使用方式）

无论你是自己用还是发给别人用，只要安装好了这个版本的 Cherry Studio（无需重新打包），只需以下简单几步即可接入服务器同步：

### 1. 找到“应用数据”目录
- 打开 Cherry Studio 客户端的 **设置** -> **数据**。
- 顶部第一行会显示你本机的**应用数据路径** (App Data Path)，例如：`/Users/username/Library/Application Support/CherryStudio`。
- 点击右侧的 `Open` 按钮在文件管理器中打开它。

### 2. 创建配置文件
- 在打开的这个“应用数据”文件夹根目录下，**新建一个纯文本文件**。
- 将文件重命名为精确的 **`cherry-sync.json`** （注意必须全是小写且后缀为 `.json`）。
- 使用代码编辑器或记事本打开它，填写入下的配置格式：
  ```json
  {
    "server": "http://你的服务器IP或域名:3456",
    "token": "你部署服务器时设置的授权Token"
  }
  ```
  *(请将这两项替换为你自己部署的 `cherry-sync-server` 的实际地址和秘钥)*

### 3. 重启生效
- 配置保存后，**彻底退出再重新打开** Cherry Studio。
- 此时同步进程不仅会在不阻塞主线程的情况下开始静默运作，它还会把配置常驻在本地配置表里。你的对话话题将在不同设备间顺畅流转！

---
*(注意：如果不配置此文件或该文件不存在，客户端将保留原版行为，没有任何对同步后端的请求泄露，确保了通用分发包原汁原味的安全性。)*
